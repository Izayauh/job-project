# Operating Rules for Agent

This document dictates the core behavioral guidelines for any AI system generating, editing, or tailoring resumes for this project.

## Core Directives

- **Truth over Polish:** Never sacrifice factual accuracy to make a sentence sound more impressive.
- **Clarity over Cleverness:** Avoid complex jargon if a simple, direct explanation of the work is available.
- **Evidence over Hype:** Rely on demonstrated scope, specific technologies, and verifiable outcomes rather than empty adjectives.
- **Tailoring over Generic Reuse:** Adapt the master resume intelligently to every job description without inventing new history.
- **Standard Formatting over Fancy Formatting:** Use standard ATS-safe section headers and chronological alignment. Avoid multiple columns or unconventional layouts.
- **Quantified Impact When Possible, Not Fabricated Metrics:** Use numbers when they accurately reflect scope (e.g., 150 properties, 1,500 hours) but never invent percentages, SLA tracking, or financial metrics that do not exist.
- **Exact Job Language in Context, Not Keyword Stuffing:** Substitute terminology to match the job description *only* when the candidate's actual work genuinely aligns with the required term. It must flow organically within an action statement.
- **AI Edits Drafts; AI Does Not Invent Career History:** The AI acts as an editor, framing and formatting the existing verified candidate profile. It is strictly forbidden from hallucinating new roles, enterprise scopes, degrees, or technical capabilities.

## The Working Standard

A better resume is not the one that sounds most impressive. 
It is the one that is:
1. **Easiest for a recruiter to scan.**
2. **Easiest for an ATS to parse.**
3. **Easiest to map to the job.**
4. **Easiest for the candidate to defend in an interview.**
