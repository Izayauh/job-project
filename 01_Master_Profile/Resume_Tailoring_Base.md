## Core Resume-Writing Rules

1. Let the projects carry the technical depth.
2. Let Nexus WiFi show customer-facing troubleshooting, equipment triage, and operational reliability.
3. Do not compensate for uncertainty by inventing technical depth.
4. Do not frame Nexus WiFi as network engineering — it is support-level triage.
5. Do not link repos or websites unless they are public, presentable, and help more than hurt.
6. No fabricated metrics, credentials, or technical claims.

## Items to Avoid Claiming

| Claim | Problem |
|-------|---------|
| Protocol-level network engineering (deep DNS config, VPN tunnel setup, routing) | Nexus WiFi was support-level triage — not enough specific recall for protocol-level claims |
| "Infrastructure monitoring and incident response" | No monitoring tools confirmed |
| "SLA-focused support delivery" | No SLA metrics available |
| "Technical documentation" | No specific documentation artifacts confirmed |
| Any bachelor's degree | Transcript shows associate-level coursework, withdrawn |
| "200% sales increase" as personal achievement | Business grew during this period; the system supported it |
| Live Pilot as a "shipped product" | It's a functional prototype, not complete |
| Impulse as "production-stable" | Has edge-case instability, not actively maintained |
